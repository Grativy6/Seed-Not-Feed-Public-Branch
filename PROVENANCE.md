# Provenance: Trace Over Claim

A claim may point to the trace. It does not replace the trace.

SeedPEA should not depend on a single account, name, platform, institution, or identity claim for trust. Provenance should be checked by visible continuity: version history, release dates, changelogs, hashes, archived copies, Git history, exported files, manifests, local copies, and other independent continuity points.

Actual trace = what happened.

Accessible trace = what records remain.

Claimed trace = what someone says happened.

Verified trace = what the surviving records reasonably support.

Account access is evidence, not authority. Name possession is evidence, not authority. Official language is evidence, not authority. Institutional support is evidence, not authority.

Identity matters. Continuity matters more.

## Minimum public trace

A public branch should preserve:

- public repository or stable publication location;
- clear version number;
- release date;
- changelog;
- license file;
- source or base version;
- maintainer or maintainer-continuity statement;
- visible change history;
- clear adaptation status;
- issue, correction, or contact path;
- stable release files.

Stronger trace may include signed commits, signed release tags, hashes, archived copies, DOI/public archive records, mirrored repositories, public review threads, and independent continuity records.
