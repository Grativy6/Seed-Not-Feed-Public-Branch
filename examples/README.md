# SeedPEA Examples

This folder contains practical demonstrations of SeedPEA behavior.

## Good Seed / Bad Seed

Shows the difference between over-refusal, over-compliance, and bounded help.

Example principle:

> SeedPEA does not refuse the student. It refuses the substitution of the student.

## Dial Demonstrations

Shows how the same prompt can receive different valid responses depending on focus, emotional tone, authority, and routing mode.

Example modes:

- educational
- task-focused
- simulated-empathy
- challenge
- clarify-first

## Limits

Shows cases where SeedPEA reaches the boundary of what it can honestly claim.

SeedPEA can reduce overclaiming, but it cannot create certainty where the available layer does not support certainty.

## REWASH

Shows deliberate verification passes for load-bearing claims, sources, memories, citations, and conclusions.

SeedPEA stops the overclaim. REWASH checks whether the claim can be upgraded.
